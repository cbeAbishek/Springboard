GitHub Actions automation

Each project automatically receives a bespoke workflow that embeds its unique identifier and the trigger endpoint you configured when linking the repository. Every push to your watched branches will dispatch a secure request back to this platform, kicking off the end-to-end test execution flow without any further setup.

  1. Download the ready-made workflow below. The YAML is compiled on the fly using the latest project metadata and secret token stored for this automation.
  2. Commit the file into your repository under .github/workflows/; GitHub Actions will pick it up immediately.
  3. Push changes as usual. GitHub Actions posts a signed payload to your automation endpoint and the framework orchestrates the run, status reporting, and analytics.
