Quick Start Guide

1. Access the Dashboard

Open your browser and navigate to:

http://localhost:8080/

2. Start the Application

Run the Spring Boot application:

mvn spring-boot:run

API Examples

Create a Test Case

POST /api/testcases { "name": "Login Test", "description": "Test user login functionality", "testType": "WEB_UI", "priority": "HIGH", "testSuite": "Authentication", "environment": "dev", "testData": "{\"url\":\"https://example.com/login\",\"username\":\"test\",\"password\":\"password\"}", "expectedResult": "User should login successfully" }

Execute a Test Batch

POST /api/execution/batch { "testSuite": "Authentication", "environment": "dev", "parallelThreads": 2 }

Best Practices

Test Organization

  • • Group tests by functionality (Test Suite)
  • • Use descriptive test case names
  • • Set appropriate priorities
  • • Separate tests by environment

Execution Strategy

  • • Start with single test execution
  • • Use batch execution for regression
  • • Monitor execution progress
  • • Review reports after execution

Troubleshooting

⚠️ API Offline

If you see "API Offline" status:

  • 1. Ensure the Spring Boot application is running
  • 2. Check that port 8080 is not blocked
  • 3. Verify the application started without errors

📱 Mobile Device Warning

The application is optimized for desktop use:

  • • Use a desktop or laptop for best experience
  • • Minimum screen width of 1024px recommended
